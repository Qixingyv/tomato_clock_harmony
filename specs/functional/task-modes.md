# 任务模式设计

> 版本：v1.0
> 创建日期：2026-03-18
> 状态：需求确认中

---

## 2.1 概述

番茄钟应用支持两种任务模式，满足用户不同的时间管理需求：
- **待办模式**：一次性任务，完成指定番茄数后结束
- **计划模式**：每日重复任务，支持双重目标追踪

---

## 2.2 待办模式（Todo Mode）

### 2.2.1 定义
一次性任务，用户设定目标番茄数，完成后任务即结束。

### 2.2.2 适用场景
- 完成一份报告
- 学习某个章节
- 准备一次演讲
- 处理邮件

### 2.2.3 数据结构

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| mode | number | 任务类型标识 | 1 |
| name | string | 任务名称 | - |
| tomato_minutes | number | 单个番茄钟时长（分钟） | 25 |
| rest_minutes | number | 休息时长（分钟） | 5 |
| target_tomato_count | number | 目标番茄总数 | 1 |
| finished_tomato_count | number | 已完成番茄数 | 0 |
| status | number | 任务状态（1=进行中，2=已完成，5=已取消） | 1 |
| created_at | Date | 创建时间 | 当前时间 |
| updated_at | Date | 更新时间 | 当前时间 |

### 2.2.4 任务状态流转

```
[创建] → 进行中(1)
            ↓
    ┌───────┴───────────┐
    ↓                   ↓
 达成目标            用户取消
    ↓                   ↓
已完成(2)          已取消(5)
    ↑                   ↑
    └───────┬───────────┘
            ↓
        历史记录
```

### 2.2.5 状态定义

| 状态码 | 名称 | 说明 | 显示位置 |
|--------|------|------|----------|
| 1 | 进行中 | 任务未完成，显示在任务列表 | 任务列表 |
| 2 | 已完成 | 达到目标番茄数 | 历史记录 |
| 5 | 已取消 | 用户主动取消任务 | 历史记录（可筛选） |

### 2.2.6 行为规则

1. **任务创建**
   - 用户输入任务名称
   - 设置目标番茄数（默认1个）
   - 可自定义番茄时长和休息时长

2. **计时流程**
   - 每完成一个番茄钟，`finished_tomato_count` +1
   - 达到 `target_tomato_count` 时，任务状态变为"已完成"
   - 已完成任务移入历史记录

3. **取消任务**
   - 用户在任务编辑页点击"删除"按钮
   - 弹出确认对话框："确认取消此任务？取消后任务将不再显示，但可在历史记录中查看"
   - 确认后任务状态变为"已取消"
   - 已取消任务移入历史记录，可通过筛选查看

4. **任务状态**
   - 进行中：显示在任务列表
   - 已完成：显示在历史记录
   - 已取消：显示在历史记录，可恢复到进行中

## 2.3 计划模式（Plan Mode）

### 2.3.1 定义
每日重复的长期任务，用户设定"每天投入多少"和"坚持多少天"，养成习惯。

### 2.3.2 适用场景
- 每日学习英语
- 每日阅读习惯
- 每日锻炼
- 长期项目推进

### 2.3.3 数据结构

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| mode | number | 任务类型标识 | 2 |
| name | string | 任务名称 | - |
| tomato_minutes | number | 单个番茄钟时长（分钟） | 25 |
| rest_minutes | number | 休息时长（分钟） | 5 |
| daily_target_tomato_count | number | 每天目标番茄数（用户输入） | 5 |
| deadline | Date | 截止日期/坚持天数（用户输入） | null |
| today_finished_count | number | 今日已完成数 | 0 |
| finished_tomato_count | number | 累计已完成数 | 0 |
| total_target_tomato_count | number | 累计总目标（系统计算） | 计算得出 |
| last_reset_date | Date | 上次重置日期 | null |
| status | number | 任务状态（1=进行中，2=已完成，4=已过期，5=已取消） | 1 |

### 2.3.4 用户输入与系统计算

**用户输入：**
| 输入项 | 说明 | 示例 |
|--------|------|------|
| 每天目标 | 每天计划完成的番茄数 | 5个/天 |
| 坚持天数 | 计划持续的天数（通过截止日期设置） | 100天 |

**系统计算：**
```
总目标 = 每天目标 × 坚持天数
剩余天数 = 截止日期 - 今天
累计进度 = 累计已完成数 / 总目标
```

**示例：**
```
用户设置：
- 每天：5个番茄
- 坚持：100天（截止日期：100天后）

系统计算：
- 总目标：500个番茄（5 × 100）
- 剩余天数：85天
- 累计完成：75个 → 进度：15%
```

### 2.3.5 双重追踪机制

**每日目标（用户关注的核心）**
- 每天0点自动重置 `today_finished_count` 为0
- 完成每日目标后，任务标记为"今日目标已达成"
- 每日达成情况可在统计中查看
- 这是用户每天打开APP最关心的指标

**累计总目标（后台统计）**
- 系统自动计算：`total = daily_target × days`
- 记录从创建到当前的总完成番茄数
- 主要用于统计页面展示长期坚持成果
- 给用户"我已经坚持了很久"的成就感

### 2.3.6 行为规则

1. **任务创建**
   - 用户输入任务名称
   - 设置每天目标番茄数（默认5个）
   - 设置坚持天数/截止日期（可选）
   - 系统自动计算累计总目标

2. **每日重置**
   - 应用启动时检查 `last_reset_date`
   - 如果是新的一天，重置 `today_finished_count` 为0
   - 更新 `last_reset_date` 为今天

3. **计时流程**
   - 每完成一个番茄钟：
     - `today_finished_count` +1
     - `finished_tomato_count` +1
   - 达到每日目标：标记"今日目标已达成"
   - 到达截止日期：任务状态变为"已完成"

4. **任务状态**
   - 进行中：显示在任务列表
   - 今日目标已达成：继续显示，标记特殊状态
   - 已完成：到达截止日期，移入历史记录
   - 已过期：超过截止日期且未完成（如果设置了硬性截止）

5. **取消任务**
   - 用户在任务编辑页点击"删除"按钮
   - 弹出确认对话框："确认取消此任务？取消后任务将不再显示，但可在历史记录中查看"
   - 确认后任务状态变为"已取消"
   - 已取消任务移入历史记录，可通过筛选查看和恢复

### 2.3.7 状态定义

| 状态码 | 名称 | 说明 | 显示位置 |
|--------|------|------|----------|
| 1 | 进行中 | 任务未完成，显示在任务列表 | 任务列表 |
| 2 | 已完成 | 达到累计目标或到达截止日期 | 历史记录 |
| 4 | 已过期 | 超过截止日期且未完成 | 历史记录 |
| 5 | 已取消 | 用户中途取消任务 | 历史记录（可恢复） |

### 2.3.8 状态流转

```
[创建] → 进行中(1)
            ↓
    ┌───────┴───────────────────┐
    ↓                           ↓
 达成目标                    用户取消
    ↓                           ↓
已完成(2)                  已取消(5)
    ↑                           ↑
    │                      可恢复
  超期
    ↓
已过期(4)
```

**说明**：
- 已完成任务统一显示在历史记录中
- 可通过时间筛选（今日/本周/本月/更早）来管理历史任务
- MVP不单独实现"归档"功能，简化状态系统

### 2.3.9 用户心智模型

```
用户想法：我要养成学英语的习惯
    ↓
第1步：每天学多少？ → 5个番茄
    ↓
第2步：坚持多久？   → 100天
    ↓
系统：好的！总目标是500个番茄，加油！
    ↓
日常使用：今天完成5个了吗？ ✓ 今日目标达成！
    ↓
统计查看：我已经坚持了45天，累计完成225个！
```

**关键设计原则：**
- 用户只关心"每天做多少"和"坚持多久"
- 总量是系统计算的，不是用户设置的
- 用户的成就感来自"今日打卡"和"累计坚持"

## 2.4 两种模式对比

| 特性 | 待办模式 | 计划模式 |
|------|----------|----------|
| 任务性质 | 一次性 | 每日重复 |
| 目标类型 | 单一目标 | 双重目标（每日+累计） |
| 完成条件 | 达到目标番茄数 | 达到累计目标或截止日期 |
| 每日重置 | 否 | 是（今日完成数） |
| 截止日期 | 不支持 | 支持 |
| 历史记录 | 保留完成记录 | 保留每日完成记录 |

## 2.5 时长配置

### 2.5.1 配置策略
采用"全局默认 + 可选覆盖"策略：

1. **全局默认值**
   - 在设置中配置默认番茄时长和休息时长
   - 新建任务时自动填充默认值

2. **任务级覆盖**
   - 每个任务可单独设置时长
   - 覆盖后不影响全局默认值

### 2.5.2 推荐时长

| 模式 | 专注时长 | 休息时长 | 适用场景 |
|------|----------|----------|----------|
| 经典 | 25分钟 | 5分钟 | 日常任务 |
| 冲刺 | 50分钟 | 10分钟 | 深度工作 |
| 轻量 | 15分钟 | 3分钟 | 碎片时间 |

## 2.6 用户界面

### 2.6.1 任务卡片显示

**待办模式卡片**
```
┌─────────────────────────┐
│ 学习高数第三章    [待办] │
├─────────────────────────┤
│ 番茄时长: 25分钟         │
│ 休息时长: 5分钟          │
│ 已完成: 3/8个番茄        │
├─────────────────────────┤
│      [ 开始专注 ]        │
└─────────────────────────┘
```

**计划模式卡片**
```
┌─────────────────────────┐
│ 每日学习英语    [计划]   │
│ 剩余: 85天              │
├─────────────────────────┤
│ 番茄时长: 25分钟         │
│ 休息时长: 5分钟          │
│ 今日: 2/5个             │
│ 累计: 127/500个 (25%)   │
├─────────────────────────┤
│      [ 开始专注 ]        │
└─────────────────────────┘
```

**卡片内容说明：**
- **今日**：今日已完成/每日目标（用户最关心的）
- **累计**：已完成/总目标（百分比）- 给长期坚持的成就感
- **剩余天数**：截止日期 - 今天（可选，有截止日期时显示）

### 2.6.2 任务类型标签
- 待办模式：蓝色标签 "待办"
- 计划模式：绿色标签 "计划"
